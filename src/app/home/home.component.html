사용 방법

0. 면책사항

KnitBot 은 모든 기능을 무료로 제공하고 있습니다.
자동매매를 설정하여 매매하는 것은 오로지 사용자 본인의 책임입니다.
서버 통신 또는 시스템 오류 등 기술적인 문제가 발생할 수 있으며, KnitBot은 이에 대해 책임지지 않습니다.
또한 해당 사용 방법이 실제와 다를 경우에도 책임지지 않습니다.
본 사이트의 모든 기능과 관련하여 KnitBot은 일체의 책임을 지지 않으므로, 이 점을 반드시 숙지하시고 본인의 의사에 따라 사이트 이용 여부를 선택해주시기 바랍니다.

1. 텔레그램 설정

  • 생성한 봇의 Token을 입력해주세요.
    ex) 1234567890:GGA-to_LirpSuGnbdaEw_KnITbOtTOkEn
  • 본인의 chat id를 입력할 경우 다양한 기능을 사용할 수 있습니다. 선택사항.
  • 텔레그램 봇을 통하여 텔레그램 안에서 봇을 제어할 수 있는 기능입니다.

2-1. 자동매매 설정 (지표 선택)

자동매매에 활용할 지표를 선택합니다.

지표 1은 반드시 선택해야 하며, 선택한 지표에 필요한 파라미터를 직접 조절할 수 있습니다.

지표 2는 선택 사항입니다. 지표 1과 지표 2를 모두 사용할 경우, 설정한 액션은 지표 1과 지표 2가 모두 충족되어야만 작동합니다. (‘AND’ 조건으로 묶입니다.)

2-2. 봇 액션 설정

봇 액션을 설정할 수 있습니다.

‘캔들’ 은 선택한 심볼의 현재 가격을 의미합니다.
종가, 시가, 고가, 저가를 선택할 수 있으며, 기본 설정에서 세팅된 캔들 시간대를 기준으로 합니다. 기본적으로 종가를 추천드립니다.

위에서 설정한 지표 1에 대하여 가격의 상황, 상수에 대한 지표의 상황에 따른 봇의 액션을 설정할 수 있습니다.

봇의 행동의 경우, 위에 설정한 상황이 발생하였을 때, 봇이 진입을 할 것인지, 종료를 할 것 인지를 설정할 수 있습니다.

롱(숏) 설정 추가를 통해, 봇이 진입을 하는 경우와 종료를 하는 경우를 각각 설정할 수 있습니다. 진입과 종료 둘 중 하나라도 설정되지 않으면 자동매매가 불가합니다.

즉, 각각 최소 2가지 이상의 설정을 하여야 하며, 조건간 충돌이 발생해서는 안 됩니다 !

만약 롱 설정과 숏 설정의 조건이 충돌한다면 봇은 작동하지 않습니다 !

예를 들어, EMA(지표 1) 선 아래로 내려갔을 때 롱을 진입하고 싶다면, 지표 1 에 대하여 ‘캔들값이 작을 때’ 봇의 행동을 ‘진입’ 으로 설정합니다.
반대로, EMA(지표 1) 선 위로 올라갔을 때 숏을 진입하고 싶다면, 지표 1에 대하여 ‘캔들값이 클 때’ 봇의 행동을 ‘진입’으로 설정합니다.

2-3. 리버스

봇이 반대 포지션을 진입하게 되면, 이전 포지션을 자동으로 종료하는 기능입니다.
만약 이전에 봇이 롱을 진입한 경우, 아직 롱 종료 조건이 완성되지 않은 상태에서 숏의 진입 조건이 완성된다면 기존의 롱 포지션을 종료하고 숏 포지션만 진입합니다.

3. 기본 설정

거래 유형을 선택할 수 있습니다.
현재 거래유형은 선물 거래(Futures) 만 지원합니다.

거래 심볼 선택에서 봇이 거래할 심볼을 선택할 수 있습니다.

캔들 설정에서는 자동매매 지표나 봇 액션 조건을 판단할 때 기준이 되는 시간대를 설정합니다. 설정되는 캔들 시간대에 따라 특정 지표들이나 조건들이 변화하므로 주의해서 설정해야 합니다.

수량은 선택한 거래 심볼을 자동 매매할 때 들어갈 수량을 의미 합니다.

이 때 수량은 선택된 코인의 수량이며 USDT가 아니므로, 수량은 반드시 코인의 수량을 기준으로 설정하여야 합니다 !

4. 저장

모든 세팅을 완료하였다면 봇을 실행하기 전 저장해야 합니다.
저장을 하지 않는다면 봇이 실행되지 않거나 세팅이 반영되지 않는 등 오류가 발생할 수 있으니 반드시 저장하여야 합니다.

5. 봇 가동 및 중지

리모컨의 우측 아이콘을 통해 봇을 켜고 끌 수 있습니다.

6-1. API 설정하기

  • 거래소를 선택해주세요.
  • 거래소에서 발급받은 API 키를 입력해주세요. 거래소별로 API를 발급받는 방법은 아래를 참고해주세요.
  • 거래소에서 발급받은 API 비밀번호(Secret Key)를 입력해주세요. 거래소별로 단어가 약간씩 다를 수 있으니 주의해주세요.
  • 거래소에서 발급받은 API Passphase를 입력해주세요. 해당 입력란은 비트겟(Bitget) 거래소를 선택하였을 경우 나오며, 거래소 별로 없는 경우도 있습니다.

6-2. 비트겟 거래소 API 발급방법

비트겟 홈페이지에 로그인하여(1번) 우측 상단의 프로필을 클릭합니다.(2번) API Management 를 클릭해주세요.

Create new API 를 클릭해주세요.

  • API 이름을 영어로 자유롭게 적어주세요.
  • 숫자 비밀번호를 만들어주세요. 8~32자리까지 가능합니다.
  • Read-only 만 체크해주세요.
  • 인증을 진행 (4~5번) 한 후, Next를 눌러 (6번) 나온 모든 부분을 전부 복사하여 안전하게 보관해야 합니다.
